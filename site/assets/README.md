# Screenshots needed

Capture these against the live app and save as PNG at the given filename. Aim for 16:10-ish crops (browser window content area, no OS chrome) so they drop cleanly into the landing page's fixed-aspect-ratio frames.

- **hero.png** — Full app view with a plan loaded (e.g. example 22, "Cardinality Trap"), hierarchical tree view, a couple of nodes with visible badges. This is the big above-the-fold screenshot, so pick a visually rich, readable plan.
- **tree.png** — Hierarchical tree view zoomed to show the node shapes, color-coded operation categories, and the connecting edges clearly.
- **hotspot.png** — A node with the red "Hotspot" ring/badge visible, ideally with the top-5 hotspots summary panel also in frame (visible when no node is selected).
- **cardinality.png** — Node detail panel open on a node with a cardinality mismatch, showing the E-Rows vs A-Rows comparison and severity badge (warn/bad).
- **sankey.png** — Sankey diagram view of a plan with several operations, metric toggle visible (e.g. set to Rows or A-Time).
- **compare.png** — Compare view with two plans loaded side by side, showing per-node metric deltas and improvement/regression indicators.
- **annotations.png** — A node with a color highlight ring applied and its annotation note visible in the detail panel or editor.
